# ADE Windows UI 设计基线

## 设计方向

界面定位为“可信、安静、精密的专业工作台”。视觉上避免普通聊天应用的大面积气泡，也不照搬传统 IDE 的高密度黑色面板。中心始终是用户正在阅读和修改的成果，Agent 是围绕成果工作的执行者。

首版采用深色工作台：石墨黑背景、偏暖的纸张色文档面板、蓝绿色作为主操作色、珊瑚色作为风险和待处理提示。信息层级通过面板明度、间距和细边框建立，不依赖大量阴影。

## 首版关键状态

### 未打开项目

显示最近项目、新建/导入项目和三步入门说明，不显示空白文件树或无意义的聊天框。

### 项目就绪

默认打开最近文件，右侧显示可用 Agent。底部面板收起，仅保留任务运行摘要。

### 文本已选择

选区上方出现浮动工具条：`批注`、`发送给 Agent`、`解释`、`重写`。创建批注后右侧切换到批注详情，并自动生成上下文预览。

### Agent 运行中

顶部和 Agent 卡片同时显示运行状态。底部事件面板展开，但不抢走文档焦点。危险操作使用内联权限卡片，说明影响路径和恢复能力。

### 审核模式

中心切换为差异视图，右侧显示批注验收项。直接事务模式使用“保留变更/恢复变更”，隔离候选模式使用“合并/丢弃”，避免语义混淆。

## 交互规则

- 主操作一次只保留一个视觉焦点。
- Agent 自然语言、工具调用、命令和文件修改使用不同事件样式。
- 所有图标按钮提供文字提示、键盘焦点和可访问名称。
- 颜色不是唯一状态表达，同时使用图标、文字和形状。
- 文档滚动、文件树和事件流独立滚动。
- 低于 1180 px 时自动折叠项目面板；低于 900 px 时右侧面板变为抽屉。
- 支持 `Ctrl+P` 文件跳转、`Ctrl+Shift+P` 命令面板、`Ctrl+Enter` 派发任务、`Alt+A` 创建批注。
